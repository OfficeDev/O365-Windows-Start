## 概要 ##

Office 365 の起動装置プロジェクトのサンプルに対してファイル、カレンダーの基本操作を説明するために、API のOffice 365 ツールクライアント ライブラリを使用して、 事業所 365. サンプル内のサービス エンドポイントの連絡先もエクスペリエンスの 1 つのアプリケーションに複数のOffice 365サービスに対する認証をユーザーとグループのサービスからユーザー情報を取得する方法を示します。場合がこのプロジェクトの更新時に電子メールなど、複数の Api を使用する方法の例を追加して、そうすることをチェックします。

このサンプルで実行できる操作を次に示します。

予定表  
  - 予定表のイベントを取得します。  
  - イベントを作成します。  
  - 更新イベント  
  - イベントを削除します。  

連絡先  
  - 連絡先を取得します。  
  - 連絡先を作成します。  
  - 連絡先を更新します。  
  - 連絡先を削除します。  
  - 連絡先の写真を変更  

自分のファイル  
  - ファイルとフォルダーを取得します。  
  - テキスト ファイルを作成します。  
  - ファイルまたはフォルダーを削除します。  
  - テキスト ファイルの内容 (OneDrive) の読み取り  
  - テキスト ファイルの内容を更新します。  
  - ファイルをダウンロードします。  
  - ファイルをアップロードします。  

ユーザーとグループ  
  - 表示名を取得します。  
  - ジョブのタイトルを取得します。  
  - プロフィールの画像を取得します。  
  - ユーザーの ID を取得します。  
  - 入出力状態チェックに署名されました。  
	
メール  
  - (ページ単位の結果) を含むメールを取得します。  
  - メールを送信します。  
  - メールを削除します。

## 前提条件と構成 ##

このサンプルを実行するには次のものが必要です。  

  - Windows 8.1  
  - Visual Studio の更新​​3 と 2013。  
  - [ Office 365 APIツールバージョン 1.3.41104.1](http://aka.ms/k0534n)です。  
  - [ Office 365開発者向けサイト](http://aka.ms/ro9c62)の。  
  - このプロジェクトのファイルの一部を使用するのにOneDrive for Businessで、テナントは、発生したサインオンOneDrive for Businessを web ブラウザー経由で最初にセットアップをする必要があります。

###このサンプルを構成します。

次の手順を実行しますをサンプルを構成する。

   1. 視覚的の 2013年を使用して O365 Api 開始 Windows.sln ファイルを開きます。
   2. 登録し、(以下で説明)、 Office 365サービスを使用するようにアプリケーションを構成します。
   3. ソリューションをビルドします。NuGetパッケージの復元機能は、packages.config ファイルに記載されているアセンブリを読み込みます。これは、一部のアセンブリの古いバージョンを取得しないように、次の手順で接続されているサービスを追加する前に行ってください。

###Office 365の Api を使用するアプリケーションを登録します。

これは (これは、登録プロセスを自動化する) を視覚的のOffice 365 API のツールを使用して実行できます。ダウンロードし、 視覚的のｷﾞｬﾗﾘｰから、 Office 365の API ツールをインストールしてください。

   1. ソリューションエクスプローラーローラー] ウィンドウで Office365Starter のプロジェクト -> を選択します ->接続済みのサービスを追加します。
   2. サービス管理職 ] ダイアログ ボックスが表示されます。Office 365と通風調節装置アプリを選択します。
   3. サインイン ダイアログ ボックスで、Office 365 テナントのユーザー名とパスワードを入力します。Office 365 開発者サイトを使用することをお勧めします。多くの場合、このユーザー名はパターン<your-name><tenant-name>@。 onmicrosoft.com。開発者サイトがあるない場合、MSDN 特典の一部として、無償の開発者向けサイトを取得または無償試用版にサインアップできます。ユーザーが、テナント管理者ユーザーをする必要がありますに注意してください-のテナントは、Office 365 開発者向けサイトの一部として作成し、これは場合に、既に可能性があります。また開発者のアカウントは通常 1 つのサインイン</tenant-name></your-name>
   4. サインインしているすべてのサービスの一覧が表示されます。最初に、アクセス許可が選択されていません、まだすべてのサービスを利用するアプリケーションが登録されていないよう。 
   5. このサンプルで使用するサービスを登録するには、次のアクセス許可を選択します。  
   	- (予定表)-ユーザーの予定表へのフル アクセスがあります。  
	- (連絡先) であるユーザーの連絡先へのフル アクセス  
	- (ファイル): の編集や、ユーザーのファイルを削除します。  
	- (ユーザーおよびグループ) – サイン ・ オンを有効にして、ユーザーのプロファイルの読み取り  
	- (ユーザーおよびグループ)-組織のディレクトリへのアクセスします。  
	- (メール) の読み取りと書き込みのアクセス権をユーザーのメール
	- (メール) のユーザーとしてメールを送信
  
   6. サービス管理職 ] ダイアログ ボックスで [ok] をクリックします。

**メモ:**段階 6 では、たとえば、パッケージのインストール中にエラーが表示された場合 * が見つかりません"Microsoft.Azure.ActiveDirectory.GraphClient"* では、ソリューションを配置する場所のローカル パスがいない確認すぎる時間が長いと深い。この問題を解決するドライブのルートに近いソリューションを移動します。私たちも今後のアップデートでフォルダー名を短くを操作します。

## トラブルシューティング ##



- すなわち、誰かが特権を持たないでグローバル管理者ではない、通常のユーザーとしてOffice 365に接続すると、「が不足している特権」例外が表示されます。接続されているサービスを追加すると*、組織のディレクトリのアクセス*許可を設定したかどうかを確認します。




- パッケージのインストール中に、エラーを受信 * 見つかりません"Microsoft.Azure.ActiveDirectory.GraphClient"*。ソリューションを配置する場所のローカル パスがいない確認すぎる時間が長いと深いです。この問題を解決するドライブのルートに近いソリューションを移動します。私たちも今後のアップデートでフォルダー名を短くを操作します。  



- 展開して、アプリケーションを[Windows のプライバシー設定](http://aka.ms/gqqx6p)] メニューで、アカウント情報にアクセスする機能を持っていない場合を実行すると、認証エラーに実行できます。**上**に**自分の名前、画像、およびその他のアカウント情報にアクセスするアプリケーションの使用**を設定します。この設定は、Windows Update によってリセットできます。 

- インストールされているアプリケーションに対して、Windows アプリ認定キットを実行して、アプリケーションがサポートされている Api テストに失敗します。これは、場合、Visual Studio のツールの一部のアセンブリの古いバージョンをインストールするためが発生しました。Microsoft.Azure.ActiveDirectory.GraphClient と、Microsoft.OData アセンブリをプロジェクトの packages.config ファイル内のエントリを確認します。これらのアセンブリのバージョン番号が[packages.config のこのリポジトリのバージョン](https://github.com/OfficeDev/O365-Windows-Start/blob/master/Office365StarterProject/packages.config)では、バージョン番号と一致するかどうかを確認します。再構築すると、更新済みのアセンブリを使用して、ソリューションを再インストール、アプリケーションは、サポートされている Api テストを渡す必要があります。
